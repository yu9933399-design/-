# 美食点餐系统 — 软件测试报告

**项目名称**：美食点餐系统（JSP + Servlet + MySQL）  
**测试日期**：2026-06-14  
**测试人员**：资深Web软件测试工程师  
**测试范围**：功能测试、安全测试、异常场景测试、边界测试

---

## 一、系统架构概述

| 层级 | 技术栈 | 文件数 |
|------|--------|--------|
| 视图层 | JSP + Bootstrap 5 + JSTL | 18个JSP |
| 控制层 | Servlet（注解映射） | 17个Servlet |
| 业务层 | Java Service | 5个Service |
| 数据层 | DAO + Druid连接池 + Commons DbUtils | 6个DAO |
| 数据库 | MySQL 8.0（utf8mb4） | 6张表 |
| 过滤器 | EncodingFilter + LoginFilter + AdminFilter | 3个 |

**角色体系**：普通用户（role=0）、管理员（role=1）、商家（role=2）  
**订单状态**：待支付(0) → 配送中(1/2) → 已完成(3)  
**用户状态**：正常(0)、禁用(1)、待审核(2)

---

## 四、整体测评结论

### 评分

| 维度 | 评分（/10） | 说明 |
|------|-------------|------|
| 功能完整性 | 7 | 核心流程齐全，但缺少订单取消、退款、评价等功能 |
| 代码质量 | 5 | 分层清晰但存在大量静默异常捕获，DAO层连接管理不一致 |
| 安全性 | 3 | 存在IDOR、XSS、CSRF等OWASP Top 10漏洞 |
| 健壮性 | 4 | 服务端缺少大量输入校验，异常直接透出数据库错误信息 |
| 性能 | 5 | 连接池配置合理，但缺少分页和查询优化 |
| 用户体验 | 6 | 前端交互较流畅，但错误提示不友好 |

**综合评分：5.0 / 10**

### 系统优点

1. **分层架构清晰**：Controller → Service → DAO 分层明确，职责分离合理
2. **密码安全**：使用 BCrypt 哈希存储密码，安全性较好
3. **乐观锁防超卖**：`UPDATE dish SET stock = stock - ? WHERE stock >= ?` 有效防止库存超卖
4. **事务管理**：`OrderService.submitOrder()` 使用手动事务 + 回滚，保障订单原子性
5. **角色权限分离**：三角色体系（用户/商家/管理员）设计合理
6. **连接池**：使用 Druid 连接池，配置了合理的初始连接和最大连接数
7. **订单号生成**：时间戳+序号格式，可读性好
8. **前端交互**：Bootstrap 5 响应式布局，AJAX无刷新操作购物车

### 系统缺点

1. **安全防护严重不足**：无 CSRF、有 XSS、IDOR越权、无 HTTPS
2. **输入校验形同虚设**：几乎全部依赖前端 required 属性，后端无校验
3. **异常处理粗放**：大量 `catch (Exception e) { e.printStackTrace(); }` 或 `catch (Exception e) {}`
4. **缺少关键功能**：无订单取消、无退款、无评价、无数据统计图表
5. **性能隐患**：列表页无分页、AdminFilter每次全表扫描
6. **Session安全**：未做 Session Fixation 防护
7. **硬编码配置**：数据库密码明文写在 properties 文件中
