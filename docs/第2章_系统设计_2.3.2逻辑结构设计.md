# 第2章 系统设计（续）

### 2.3.2 逻辑结构设计

根据概念结构设计阶段得到的E-R图，将实体及实体间的关系转化为关系模式。以下列出本系统全部11张数据表的关系模式，主键以下划线标注，外键标注参照关系。

---

#### 表1：用户表（user）

**关系模式：**

user(<u>user_id</u>, username, password, real_name, phone, email, address, role, status, shop_category)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>user_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 用户ID，主键 |
| username | VARCHAR(30) | UNIQUE NOT NULL | 用户名，唯一 |
| password | VARCHAR(255) | NOT NULL | BCrypt加密密码 |
| real_name | VARCHAR(20) | | 真实姓名（商家时为店铺名称） |
| phone | VARCHAR(15) | | 手机号码 |
| email | VARCHAR(50) | | 电子邮箱 |
| address | VARCHAR(200) | | 收货地址 |
| role | TINYINT | NOT NULL DEFAULT 0 | 0=普通用户, 1=管理员, 2=商家 |
| status | TINYINT | NOT NULL DEFAULT 0 | 0=正常, 1=禁用, 2=审核中 |
| shop_category | VARCHAR(20) | DEFAULT NULL | 店铺品类 |

---

#### 表2：菜品分类表（category）

**关系模式：**

category(<u>category_id</u>, category_name, sort_order)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>category_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 分类ID，主键 |
| category_name | VARCHAR(30) | NOT NULL | 分类名称 |
| sort_order | INT | DEFAULT 0 | 排序值 |

---

#### 表3：菜品表（dish）

**关系模式：**

dish(<u>dish_id</u>, merchant_id, dish_name, price, image_url, description, stock, category_id, status, is_recommend, special_price, special_start, special_end)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>dish_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 菜品ID，主键 |
| merchant_id | INT | NOT NULL | FK → **user.user_id**（商家ID） |
| dish_name | VARCHAR(50) | NOT NULL | 菜品名称 |
| price | DECIMAL(10,2) | NOT NULL | 菜品原价 |
| image_url | VARCHAR(200) | | 图片路径 |
| description | VARCHAR(500) | | 菜品描述 |
| stock | INT | NOT NULL | 库存数量 |
| category_id | INT | NOT NULL | FK → **category.category_id**（分类ID） |
| status | TINYINT | DEFAULT 1 | 1=上架, 0=下架 |
| is_recommend | TINYINT | DEFAULT 0 | 1=推荐, 0=普通 |
| special_price | DECIMAL(10,2) | DEFAULT NULL | 特价，NULL表示无特价 |
| special_start | DATETIME | DEFAULT NULL | 特价开始时间 |
| special_end | DATETIME | DEFAULT NULL | 特价结束时间 |

> 参照完整性约束：merchant_id参照user表的user_id，保证菜品必须归属于已存在的商家；category_id参照category表的category_id，保证菜品必须归属于已存在的分类。

---

#### 表4：购物车表（cart）

**关系模式：**

cart(<u>cart_id</u>, user_id, dish_id, quantity, add_time)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>cart_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 购物车ID，主键 |
| user_id | INT | NOT NULL | FK → **user.user_id**（用户ID） |
| dish_id | INT | NOT NULL | FK → **dish.dish_id**（菜品ID） |
| quantity | INT | NOT NULL | 购买数量 |
| add_time | DATETIME | NOT NULL | 加入时间 |

> 参照完整性约束：user_id参照user表，dish_id参照dish表。购物车通过dish_id间接关联merchant_id，实现按店铺分组。

---

#### 表5：订单表（order）

**关系模式：**

order(<u>order_id</u>, order_no, user_id, shop_id, receiver_name, receiver_phone, receiver_address, total_amount, payment_method, order_status, create_time, pay_time, accept_time, cancel_reason, delivery_status, delivery_position)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>order_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 订单ID，主键 |
| order_no | VARCHAR(20) | UNIQUE NOT NULL | 订单编号 |
| user_id | INT | NOT NULL | FK → **user.user_id**（下单用户） |
| shop_id | INT | DEFAULT NULL | FK → **user.user_id**（所属商家） |
| receiver_name | VARCHAR(20) | NOT NULL | 收货人姓名 |
| receiver_phone | VARCHAR(15) | NOT NULL | 收货人电话 |
| receiver_address | VARCHAR(200) | NOT NULL | 收货地址 |
| total_amount | DECIMAL(10,2) | NOT NULL | 订单总金额 |
| payment_method | VARCHAR(20) | NOT NULL | 支付方式 |
| order_status | TINYINT | NOT NULL DEFAULT 1 | 订单状态码 |
| create_time | DATETIME | NOT NULL | 下单时间 |
| pay_time | DATETIME | DEFAULT NULL | 支付时间 |
| accept_time | DATETIME | DEFAULT NULL | 商家接单时间 |
| cancel_reason | VARCHAR(500) | DEFAULT NULL | 取消原因 |
| delivery_status | TINYINT | NOT NULL DEFAULT 0 | 0=待配送, 1=配送中, 2=已送达 |
| delivery_position | VARCHAR(500) | DEFAULT NULL | 配送员当前位置描述 |

> 参照完整性约束：user_id参照user表（下单用户），shop_id参照user表（所属商家）。order_status取值范围为{1,2,4,5,6,8,9}，分别对应待支付、待接单、已接单、制作中、配送中、已完成、已取消。

---

#### 表6：订单项表（order_item）

**关系模式：**

order_item(<u>item_id</u>, order_id, dish_id, dish_name, price, quantity, subtotal)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>item_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 订单项ID，主键 |
| order_id | INT | NOT NULL | FK → **order.order_id**（所属订单） |
| dish_id | INT | NOT NULL | FK → **dish.dish_id**（菜品ID） |
| dish_name | VARCHAR(50) | NOT NULL | 菜品名称（下单时快照） |
| price | DECIMAL(10,2) | NOT NULL | 菜品单价（下单时快照） |
| quantity | INT | NOT NULL | 购买数量 |
| subtotal | DECIMAL(10,2) | NOT NULL | 小计金额 |

> 参照完整性约束：order_id参照order表，dish_id参照dish表。dish_name和price为下单时刻的价格快照，与dish表当前值独立，保证历史订单数据一致性。

---

#### 表7：订单状态变更日志表（order_log）

**关系模式：**

order_log(<u>log_id</u>, order_id, from_status, to_status, operator_id, operator_role, reason, create_time)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>log_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 日志ID，主键 |
| order_id | INT | NOT NULL | FK → **order.order_id**（关联订单） |
| from_status | TINYINT | DEFAULT NULL | 变更前状态 |
| to_status | TINYINT | NOT NULL | 变更后状态 |
| operator_id | INT | NOT NULL | 操作人ID |
| operator_role | TINYINT | NOT NULL | 0=用户, 1=管理员, 2=商家 |
| reason | VARCHAR(500) | DEFAULT NULL | 操作原因 |
| create_time | DATETIME | NOT NULL | 操作时间 |

> 参照完整性约束：order_id参照order表。operator_role标识操作人身份，与User表的role字段含义一致。

---

#### 表8：订单取消日志表（order_cancel_log）

**关系模式：**

order_cancel_log(<u>cancel_id</u>, order_id, cancel_reason, cancel_by, cancel_time)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>cancel_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 取消日志ID，主键 |
| order_id | INT | NOT NULL | FK → **order.order_id**（关联订单） |
| cancel_reason | VARCHAR(500) | DEFAULT NULL | 取消原因 |
| cancel_by | INT | DEFAULT NULL | 操作人ID |
| cancel_time | DATETIME | DEFAULT NULL | 取消时间 |

> 参照完整性约束：order_id参照order表。该表专门记录订单取消操作的详细信息，与order_log形成补充。

---

#### 表9：轮播图表（banner）

**关系模式：**

banner(<u>banner_id</u>, title, subtitle, image_url, link_url, sort_order, status, create_time)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>banner_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 轮播图ID，主键 |
| title | VARCHAR(100) | DEFAULT NULL | 标题文字 |
| subtitle | VARCHAR(200) | DEFAULT NULL | 副标题文字 |
| image_url | VARCHAR(500) | NOT NULL | 图片存储路径 |
| link_url | VARCHAR(500) | DEFAULT NULL | 点击跳转链接 |
| sort_order | INT | DEFAULT 0 | 排序值 |
| status | TINYINT | NOT NULL DEFAULT 1 | 1=启用, 0=禁用 |
| create_time | DATETIME | NOT NULL | 创建时间 |

> 说明：该表为独立实体，不与其他表存在外键关联。由管理员维护，用于首页轮播展示。

---

#### 表10：消息表（message）

**关系模式：**

message(<u>message_id</u>, sender_id, receiver_id, content, is_read, create_time)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>message_id</u> | INT | PRIMARY KEY AUTO_INCREMENT | 消息ID，主键 |
| sender_id | INT | NOT NULL | FK → **user.user_id**（发送者） |
| receiver_id | INT | NOT NULL | FK → **user.user_id**（接收者） |
| content | VARCHAR(1000) | NOT NULL | 消息内容 |
| is_read | TINYINT | NOT NULL DEFAULT 0 | 0=未读, 1=已读 |
| create_time | DATETIME | NOT NULL | 发送时间 |

> 参照完整性约束：sender_id和receiver_id均参照user表。同一条消息同时关联发送方和接收方两个用户，实现用户间的双向消息通信。

---

#### 表11：数据库版本管理表（schema_version）

**关系模式：**

schema_version(<u>version</u>, description, applied_at)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| <u>version</u> | VARCHAR(50) | PRIMARY KEY | 版本号 |
| description | VARCHAR(200) | DEFAULT NULL | 版本描述 |
| applied_at | DATETIME | DEFAULT NULL | 执行时间 |

> 说明：该表用于数据库迁移脚本的版本追踪，记录每次DDL变更的执行情况，非业务实体。

---

#### 关系模式总览

| 序号 | 关系模式 | 主键 | 外键 | 参照表 |
|------|----------|------|------|--------|
| 1 | user(<u>user_id</u>, username, password, ...) | user_id | — | — |
| 2 | category(<u>category_id</u>, category_name, ...) | category_id | — | — |
| 3 | dish(<u>dish_id</u>, merchant_id, ...) | dish_id | merchant_id | user.user_id |
| 4 | dish(...) | — | category_id | category.category_id |
| 5 | cart(<u>cart_id</u>, user_id, dish_id, ...) | cart_id | user_id | user.user_id |
| 6 | cart(...) | — | dish_id | dish.dish_id |
| 7 | order(<u>order_id</u>, order_no, user_id, ...) | order_id | user_id | user.user_id |
| 8 | order(...) | — | shop_id | user.user_id |
| 9 | order_item(<u>item_id</u>, order_id, dish_id, ...) | item_id | order_id | order.order_id |
| 10 | order_item(...) | — | dish_id | dish.dish_id |
| 11 | order_log(<u>log_id</u>, order_id, ...) | log_id | order_id | order.order_id |
| 12 | order_cancel_log(<u>cancel_id</u>, order_id, ...) | cancel_id | order_id | order.order_id |
| 13 | banner(<u>banner_id</u>, title, ...) | banner_id | — | — |
| 14 | message(<u>message_id</u>, sender_id, receiver_id, ...) | message_id | sender_id | user.user_id |
| 15 | message(...) | — | receiver_id | user.user_id |
| 16 | schema_version(<u>version</u>, ...) | version | — | — |

**外键约束汇总：**

| 外键字段 | 所在表 | 参照表.主键 | 关系类型 |
|----------|--------|------------|----------|
| dish.merchant_id | dish | user.user_id | 多对一（菜品→商家） |
| dish.category_id | dish | category.category_id | 多对一（菜品→分类） |
| cart.user_id | cart | user.user_id | 多对一（购物车→用户） |
| cart.dish_id | cart | dish.dish_id | 多对一（购物车→菜品） |
| order.user_id | order | user.user_id | 多对一（订单→用户） |
| order.shop_id | order | user.user_id | 多对一（订单→商家） |
| order_item.order_id | order_item | order.order_id | 多对一（订单项→订单） |
| order_item.dish_id | order_item | dish.dish_id | 多对一（订单项→菜品） |
| order_log.order_id | order_log | order.order_id | 多对一（日志→订单） |
| order_cancel_log.order_id | order_cancel_log | order.order_id | 多对一（取消日志→订单） |
| message.sender_id | message | user.user_id | 多对一（消息→发送者） |
| message.receiver_id | message | user.user_id | 多对一（消息→接收者） |

> 共计 **12个外键约束**，分布在7张业务表中。user表和category表为独立实体，无外键依赖；schema_version表为系统管理表，无外键依赖。
