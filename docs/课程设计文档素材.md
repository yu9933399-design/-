# 美食点餐系统 — 课程设计文档素材

## 一、项目技术栈与架构

### 1.1 技术栈
| 类别 | 技术 | 版本 |
|------|------|------|
| 后端语言 | Java | 1.8 (JDK 8) |
| Web框架 | Servlet + JSP | Servlet 4.0 |
| 数据库 | MySQL | 8.0 |
| 连接池 | Druid | 1.2.8 |
| JDBC工具 | Apache Commons DBUtils | 1.7 |
| 前端框架 | Bootstrap | 5.3.0 |
| JS库 | jQuery | 3.7.0 |
| 图标库 | Font Awesome | 6.4.0 |
| 密码加密 | BCrypt (jbcrypt) | 0.4 |
| 服务器 | Apache Tomcat | 9.0.115 |
| AI接口 | 通义千问 DashScope | qwen3.6-flash |
| IDE | Eclipse EE | - |

### 1.2 MVC三层架构
```
浏览器 → JSP(视图) → Servlet(控制器) → Service(业务逻辑) → DAO(数据访问) → MySQL
```

- **视图层(V)**：JSP + Bootstrap 5 + jQuery + CSS，共37个JSP页面
- **控制层(C)**：Servlet + Filter，共34个URL映射
- **业务层(S)**：Service类，共7个业务服务
- **数据层(M)**：DAO + Druid连接池 + JDBC，共9个DAO + 自定义CamelCaseRowProcessor

### 3.2 核心业务规则

#### 订单状态机(美团同款)
```
用户下单 → [1]待支付 → 支付 → [2]待商家接单
                                        ↓
                              商家接单 → [4]已接单 → 开始制作 → [5]制作中 → 出餐 → [6]配送中
                                        ↓                                           ↓
                              商家拒单 → [9]已取消                            确认收货 → [8]已完成
                              
用户可在[2]待接单直接取消
用户可在[4]已接单1分钟内取消，超过需申请
[5]制作中、[6]配送中用户不可取消
[8]已完成、[9]已取消不可操作
管理员可强制修改任何状态(需填原因，记录日志)
```

#### 进店点餐规则(对标美团)
- 首页所有菜品仅可点击跳转店铺，无加购按钮
- 加购/结算仅在商家店铺页内操作
- 购物车商品绑定所属店铺shopId
- 下单时校验跨店，不同店铺商品不可一起结算
- 商家订单自动按shopId过滤

#### 配送功能
- delivery_status: 0=待配送, 1=配送中, 2=已送达
- 商家/管理员可更新配送位置
- 已送达后不可再修改
- 用户仅可查看配送信息

#### AI智能客服
- 对接通义千问DashScope API (qwen3.6-flash)
- 自动读取用户全部订单(状态/金额/地址/配送信息)
- 仅回答订单、配送、取消规则相关问题
- 系统提示词约束AI行为
