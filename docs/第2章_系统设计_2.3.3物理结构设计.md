# 第2章 系统设计（续）

### 2.3.3 物理结构设计

本系统采用 **MySQL 8.0** 关系型数据库管理系统，数据库名称为 `online_order_db`，字符集统一使用 `utf8mb4`，排序规则为 `utf8mb4_general_ci`。数据库连接采用阿里巴巴 Druid 连接池进行管理，配置初始连接数5、最大连接数10、最大等待时间3秒，通过 `druid.properties` 文件统一配置。系统共创建 **11张数据表**，以下依次列出各表的物理结构。

---

#### 表1 user（用户表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | user_id | INT | 用户ID，主键，自增 | 否 |
| 2 | username | VARCHAR(30) | 用户名，唯一 | 否 |
| 3 | password | VARCHAR(255) | 登录密码（BCrypt加密） | 否 |
| 4 | real_name | VARCHAR(20) | 真实姓名（商家为店铺名） | 是 |
| 5 | phone | VARCHAR(15) | 手机号码 | 是 |
| 6 | email | VARCHAR(50) | 电子邮箱 | 是 |
| 7 | address | VARCHAR(200) | 收货地址 | 是 |
| 8 | role | TINYINT | 角色：0用户/1管理员/2商家 | 否 |
| 9 | status | TINYINT | 状态：0正常/1禁用/2审核中 | 否 |
| 10 | shop_category | VARCHAR(20) | 店铺所属品类 | 是 |

> 存储引擎：InnoDB。该表为系统核心用户表，通过role字段区分三种角色，通过status字段控制账号状态。shop_category字段仅当role=2（商家）时有效，取值为8个预设分类之一。

---

#### 表2 category（菜品分类表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | category_id | INT | 分类ID，主键，自增 | 否 |
| 2 | category_name | VARCHAR(30) | 分类名称 | 否 |
| 3 | sort_order | INT | 排序值，越小越靠前 | 是 |

> 存储引擎：InnoDB。该表存储菜品分类体系，预设8个分类：快餐简餐、奶茶饮品、川湘辣味、粤菜海鲜、甜品小吃、早餐早点、面食粥品、轻食沙拉。

---

#### 表3 dish（菜品表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | dish_id | INT | 菜品ID，主键，自增 | 否 |
| 2 | merchant_id | INT | 商家ID，外键→user.user_id | 否 |
| 3 | dish_name | VARCHAR(50) | 菜品名称 | 否 |
| 4 | price | DECIMAL(10,2) | 菜品原价 | 否 |
| 5 | image_url | VARCHAR(200) | 菜品图片路径 | 是 |
| 6 | description | VARCHAR(500) | 菜品描述 | 是 |
| 7 | stock | INT | 库存数量 | 否 |
| 8 | category_id | INT | 分类ID，外键→category.category_id | 否 |
| 9 | status | TINYINT | 上下架：1上架/0下架 | 是 |
| 10 | is_recommend | TINYINT | 是否推荐：1是/0否 | 是 |
| 11 | special_price | DECIMAL(10,2) | 特价，NULL表示无特价 | 是 |
| 12 | special_start | DATETIME | 特价开始时间 | 是 |
| 13 | special_end | DATETIME | 特价结束时间 | 是 |

> 存储引擎：InnoDB。外键约束：merchant_id关联user表（商家），category_id关联category表（分类）。special_price为NULL时表示该菜品不参与特价活动。

---

#### 表4 cart（购物车表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | cart_id | INT | 购物车ID，主键，自增 | 否 |
| 2 | user_id | INT | 用户ID，外键→user.user_id | 否 |
| 3 | dish_id | INT | 菜品ID，外键→dish.dish_id | 否 |
| 4 | quantity | INT | 购买数量 | 否 |
| 5 | add_time | DATETIME | 加入购物车时间 | 否 |

> 存储引擎：InnoDB。外键约束：user_id关联user表，dish_id关联dish表。购物车通过dish_id间接关联merchant_id实现按店铺分组。

---

#### 表5 order（订单表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | order_id | INT | 订单ID，主键，自增 | 否 |
| 2 | order_no | VARCHAR(20) | 订单编号，唯一 | 否 |
| 3 | user_id | INT | 下单用户ID，外键→user.user_id | 否 |
| 4 | shop_id | INT | 所属店铺ID，外键→user.user_id | 是 |
| 5 | receiver_name | VARCHAR(20) | 收货人姓名 | 否 |
| 6 | receiver_phone | VARCHAR(15) | 收货人电话 | 否 |
| 7 | receiver_address | VARCHAR(200) | 收货地址 | 否 |
| 8 | total_amount | DECIMAL(10,2) | 订单总金额 | 否 |
| 9 | payment_method | VARCHAR(20) | 支付方式 | 否 |
| 10 | order_status | TINYINT | 订单状态码 | 否 |
| 11 | create_time | DATETIME | 下单时间 | 否 |
| 12 | pay_time | DATETIME | 支付时间 | 是 |
| 13 | accept_time | DATETIME | 商家接单时间 | 是 |
| 14 | cancel_reason | VARCHAR(500) | 取消原因 | 是 |
| 15 | delivery_status | TINYINT | 配送状态：0待配送/1配送中/2已送达 | 是 |
| 16 | delivery_position | VARCHAR(500) | 配送员当前位置描述 | 是 |

> 存储引擎：InnoDB。order_status取值：1待支付、2待接单、4已接单、5制作中、6配送中、8已完成、9已取消。shop_id将订单绑定到具体商家，实现商家订单隔离。accept_time用于用户1分钟内取消窗口判断。

---

#### 表6 order_item（订单项表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | item_id | INT | 订单项ID，主键，自增 | 否 |
| 2 | order_id | INT | 订单ID，外键→order.order_id | 否 |
| 3 | dish_id | INT | 菜品ID，外键→dish.dish_id | 否 |
| 4 | dish_name | VARCHAR(50) | 菜品名称（下单时快照） | 否 |
| 5 | price | DECIMAL(10,2) | 菜品单价（下单时快照） | 否 |
| 6 | quantity | INT | 购买数量 | 否 |
| 7 | subtotal | DECIMAL(10,2) | 小计金额 | 否 |

> 存储引擎：InnoDB。dish_name和price为下单时刻的价格快照，即使菜品后续改名或调价，历史订单仍保留原始数据。

---

#### 表7 order_log（订单状态变更日志表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | log_id | INT | 日志ID，主键，自增 | 否 |
| 2 | order_id | INT | 订单ID，外键→order.order_id | 否 |
| 3 | from_status | TINYINT | 变更前状态 | 是 |
| 4 | to_status | TINYINT | 变更后状态 | 否 |
| 5 | operator_id | INT | 操作人ID | 否 |
| 6 | operator_role | TINYINT | 操作人角色：0用户/1管理员/2商家 | 否 |
| 7 | reason | VARCHAR(500) | 操作原因 | 是 |
| 8 | create_time | DATETIME | 操作时间 | 否 |

> 存储引擎：InnoDB。记录每次订单状态变更的完整审计信息，支持管理员查看订单操作历史。

---

#### 表8 order_cancel_log（订单取消日志表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | cancel_id | INT | 取消日志ID，主键，自增 | 否 |
| 2 | order_id | INT | 订单ID，外键→order.order_id | 否 |
| 3 | cancel_reason | VARCHAR(500) | 取消原因 | 是 |
| 4 | cancel_by | INT | 操作人ID | 是 |
| 5 | cancel_time | DATETIME | 取消时间 | 是 |

> 存储引擎：InnoDB。专门记录订单取消操作的详细信息，与order_log形成互补。

---

#### 表9 banner（轮播图表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | banner_id | INT | 轮播图ID，主键，自增 | 否 |
| 2 | title | VARCHAR(100) | 标题文字 | 是 |
| 3 | subtitle | VARCHAR(200) | 副标题文字 | 是 |
| 4 | image_url | VARCHAR(500) | 图片存储路径 | 否 |
| 5 | link_url | VARCHAR(500) | 点击跳转链接 | 是 |
| 6 | sort_order | INT | 排序值，越小越靠前 | 是 |
| 7 | status | TINYINT | 启停：1启用/0禁用 | 否 |
| 8 | create_time | DATETIME | 创建时间 | 否 |

> 存储引擎：InnoDB。由管理员通过后台管理页面维护，支持图片文件上传。

---

#### 表10 message（消息表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | message_id | INT | 消息ID，主键，自增 | 否 |
| 2 | sender_id | INT | 发送者ID，外键→user.user_id | 否 |
| 3 | receiver_id | INT | 接收者ID，外键→user.user_id | 否 |
| 4 | content | VARCHAR(1000) | 消息内容 | 否 |
| 5 | is_read | TINYINT | 已读状态：0未读/1已读 | 否 |
| 6 | create_time | DATETIME | 发送时间 | 否 |

> 存储引擎：InnoDB。sender_id和receiver_id同时关联user表，实现用户间双向消息通信。支持用户-商家、商家-管理员、用户-管理员三种消息通道。

---

#### 表11 schema_version（数据库版本管理表）

| 编号 | 字段名称 | 数据类型 | 说明 | 允许空 |
|------|----------|----------|------|--------|
| 1 | version | VARCHAR(50) | 版本号 | 否 |
| 2 | description | VARCHAR(200) | 版本描述 | 是 |
| 3 | applied_at | DATETIME | 执行时间 | 是 |

> 存储引擎：InnoDB。用于数据库迁移脚本的版本追踪，记录每次DDL变更的执行情况，非业务实体。
